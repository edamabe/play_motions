
# ユーザー向けインタラクティブ知育アプリ 開発要件定義書

あなたは優秀なPythonエンジニアです。以下の要件定義に従って、3歳の子供がWebカメラとマイクを使って遊べるインタラクティブなWindows用デスクトップアプリを開発してください。
拡張性とメンテナンス性を最重視し、トリガーとエフェクトのルールはメインロジックから分離した設計にしてください。

## 1. プロジェクト概要
ユーザー（子供）のWebカメラに映った「ポーズ」や、マイクから入力された「音声」をリアルタイムに検知し、それに応じて画面上に画像エフェクトを表示し、効果音を鳴らすアプリケーション。

## 2. 技術スタック
* **言語:** Python 3.x
* **映像取得・描画:** `opencv-python` (cv2)
    * 画像のアルファブレンド（透過PNGの重畳描画）もOpenCVまたはPillowを併用して実装。
* **ポーズ推定:** `mediapipe` (Poseモジュール)
* **音声認識:** `vosk` および `pyaudio`
    * 遅延を防ぐため、クラウドAPIは使用せずオフライン推論を行うこと。
* **音声再生:** `pygame` (mixerモジュール)

## 3. ディレクトリ・ファイル構成案
機能追加を容易にするため、設定とロジックを分離してください。

```text
project_root/
├── main.py              # アプリケーションのエントリーポイント、OpenCVループ
├── config.py            # トリガーとエフェクトのルールを定義する辞書（変更しやすいよう独立）
├── modules/
│   ├── pose_detector.py # MediaPipeによるポーズ判定ロジック
│   ├── voice_listener.py# Voskによる音声認識ロジック（別スレッド実行）
│   └── effect_player.py # 画像の合成とPygameによる音声再生ロジック
├── assets/
│   ├── images/          # 透過PNG画像 (apple.png, sparkle.pngなど)
│   └── sounds/          # 効果音MP3/WAV (pon.mp3, sharan.mp3など)
└── models/              # Voskの日本語言語モデル配置ディレクトリ
```

## 4. 機能要件詳細

### 4.1. メイン画面表示 (UI)
* **鏡面モード:** Webカメラから取得した映像は、左右反転（`cv2.flip`）させて表示すること。
* **フルスクリーン:** 起動時はフルスクリーン表示（`cv2.setWindowProperty`）にすること。
* **安全停止:** 子供の誤操作を防ぐため、特定のキー（例：`ESC`キー）を押下した時のみ安全に終了・リソース解放する処理を入れること。

### 4.2. ポーズ検知機能
* MediaPipe Poseを使用して骨格のランドマーク座標を取得する。
* `config.py` で定義された条件式（例：右手首のY座標 < 右肩のY座標）を評価し、一致した場合にトリガーを発火させる。
* 連続発火を防ぐため、一度トリガーされたポーズは一定時間（例：2秒間）または別のポーズに戻るまで再発火しないクールダウン処理を入れること。

### 4.3. 音声認識機能
* 映像のフレームレート低下を防ぐため、Voskによる音声認識ループは `threading` を用いてメインのOpenCVループとは別のバックグラウンドスレッドで実行すること。
* 認識したテキスト（ひらがな/カタカナ/漢字）をメインスレッドに渡し、`config.py` に登録されたキーワードと部分一致したらトリガーを発火させる。
* 幼児の不明瞭な発音に対応できるよう、1つのエフェクトに対して複数のトリガーワード（例：「ぞうさん」「どうさん」）を紐付けられるデータ構造にすること。

### 4.4. エフェクト機能
* **画像描画:** トリガー発火時、指定された画像をカメラ映像の前面に合成する。表示時間は設定可能（例：3秒間）にすること。
* **音声再生:** トリガー発火時、遅延なく指定された効果音を再生する。

## 5. `config.py` のデータ構造例（実装指示）
後からプログラミング知識がなくても書き換えられるよう、以下のようなシンプルで拡張しやすい辞書・リスト形式で定義してください。

```python
# config.py のイメージ
VOICE_TRIGGERS = [
    {
        "keywords": ["りんご", "いんご"],
        "image": "assets/images/apple.png",
        "sound": "assets/sounds/pon.mp3",
        "duration": 3.0
    }
]

POSE_TRIGGERS = [
    {
        "name": "banzai",
        "condition": lambda landmarks: landmarks['left_wrist'].y < landmarks['left_shoulder'].y and landmarks['right_wrist'].y < landmarks['right_shoulder'].y,
        "image": "assets/images/sparkle.png",
        "sound": "assets/sounds/sharan.mp3",
        "duration": 2.0
    }
]
```

## 6. 開発ステップ（段階的実行のお願い）
一度に全て実装するとバグの特定が難しいため、以下のステップに分けて実装とテストを行ってください。
まずは【ステップ1】の実装方針とコードを提示し、ユーザーの確認を待ってください。

* **ステップ1:** OpenCVでのカメラ表示（左右反転・フルスクリーン・ESC終了）とディレクトリ構成の作成。
* **ステップ2:** MediaPipeの組み込みと、単純なポーズ（バンザイ等）でのコンソール出力（print）テスト。
* **ステップ3:** PygameとOpenCVによるエフェクト（画像合成・音再生）の実装と、ポーズ判定との結合。
* **ステップ4:** Voskの組み込み（別スレッド化）と、音声トリガーの結合。
```
